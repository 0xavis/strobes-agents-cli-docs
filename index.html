            A G E N T S   A I  ·  terminal client
Claude Code, but for your Strobes pentest agents.

A local, terminal-native client for Strobes Agents AI. It binds to a remote Strobes organization over MasterKey auth, streams agent runs live in a clean Ratatui UI, and runs the shell sandbox and browser on your own machine — so the agent's tools operate on your real files.

Install → Latest release Source on GitHub
Rust · Ratatui macOS · Linux · Windows MasterKey auth local shell + browser self-update

# Install

Prebuilt binaries for every platform are on the releases page. One-liners below detect your architecture, fetch the latest release, and put strobes on your PATH.

macOS / Linux

curl -fsSL https://raw.githubusercontent.com/strobes-co/strobes-agents-cli/main/install.sh | bash

Windows (PowerShell)

irm https://github.com/strobes-co/strobes-agents-cli/releases/latest/download/strobes-x86_64-pc-windows-msvc.tar.gz -OutFile s.tgz
tar -xzf s.tgz; .\strobes-x86_64-pc-windows-msvc\strobes.exe --help

Pick a binary manually

PlatformAsset
macOS (Apple Silicon)strobes-aarch64-apple-darwin.tar.gz
macOS (Intel)strobes-x86_64-apple-darwin.tar.gz
Linux (x86-64)strobes-x86_64-unknown-linux-gnu.tar.gz
Windows (x86-64)strobes-x86_64-pc-windows-msvc.tar.gz

Each ships a .sha256 checksum. Already installed? strobes update self-updates in place. Build from source with cargo build --release (needs Rust; Chrome only for browser tools).

# Quick start

# 1. Authenticate (get a MasterKey from Strobes UI → Organization → API access)
strobes login                 # interactive prompts (or pass --base-url/--org-id/--master-key)
strobes status                # ✔ connection OK · ✔ up to date

# 2. Work
strobes                       # open the chat TUI: pick a workspace → thread → chat
strobes workspaces            # list remote workspaces (+ thread counts + credits)
strobes credits               # AI credit usage, org total + per-workspace

Env vars persist to ~/.config/strobes-ai/config.json (macOS: ~/Library/Application Support/strobes-ai/config.json). Manage multiple deployments as tenants and switch with --tenant <name>.

# Commands

CommandWhat it does
chat defaultInteractive chat TUI — streams the agent run, runs its tools locally. Workspace → thread → chat selection on open.
loginConfigure credentials for a tenant — interactive prompts or --base-url / --org-id / --master-key / --deployment flags. First configured tenant becomes the default.
tenantsList configured tenants (★ marks the default).
statusShow the active profile + connectivity, and whether an update is available.
workspacesList remote workspaces with thread counts and credits consumed.
threadsList your chat threads.
creditsAI credit usage — org total + per-workspace breakdown. Scope with --workspace / --thread.
bindPick or create a workspace (--new --name), optionally --download its files locally.
pullDownload a workspace's files to a local folder.
updateSelf-update to the latest release (downloads + replaces the binary). --force to reinstall.

--version · -VPrint the version. Global --tenant (alias --profile) selects a tenant for any command.

# Features

⌁ Multi-tenant

Configure many Strobes deployments; the first becomes default. Switch per-run with --tenant. Each keeps its own creds + bindings.

◈ Live credits

Per-run, per-chat (lifetime), per-workspace and org-wide AI credit + token usage — in the status bar and the credits command.

⊞ Workspaces

Create from the picker (prompts for a name + setup prompt → opens the AI setup chat), bind, and sync files locally. Per-workspace credit & thread counts.

⌨ Local shell sandbox

The agent's execute_command / execute_code run on your machine against the synced workspace files — Python, Bash, Node, Ruby.

◐ Local browser

9 browser_* tools drive a real local Chrome via CDP. Auto-detects Chrome, guides install, or auto-downloads Chrome for Testing.

⟳ Auto-reconnect

The WebSocket self-heals — on a network drop it re-dials with capped backoff instead of sitting idle, transparently to the UI.

⬆ Self-update

strobes update replaces the binary in place. The CLI also suggests updates on start and in status.

⌖ Fast pickers

Type-to-search workspace/thread pickers, bottom-anchored, single alternate-screen (no flicker). Esc navigates back instead of quitting.

⚑ Cross-platform

macOS (arm64/x64), Linux, and Windows binaries from CI. Native shell per OS (cmd.exe on Windows).

⎘ Copy & select

Native terminal selection works (mouse isn't captured); ^Y copies the whole transcript to your clipboard.

⚡ Fast startup

Startup round-trips (REST + WebSocket) run in parallel; a dev DNS bypass skips slow .local mDNS lookups.

◆ Clean transcript

Agent headers, tool calls + results, dimmed thinking, Markdown (incl. tables), padded layout, full thread history on open.

Browser tools need Chrome/Chromium. Set STROBES_AI_BROWSER_HEADLESS=1 for headless, STROBES_AI_CHROME=/path for a custom install, or STROBES_AI_BROWSER_AUTOINSTALL=1 to fetch a self-contained Chrome for Testing on first use.

# How it works

┌──────────────┐ pulse WS (chat + tool.local_execute) ┌──────────────────┐ │ strobes │ ◀──────────────────────────────────────▶ │ Strobes backend │ │ (local) │ workspace files · findings · approvals │ remote agents, │ └──────────────┘ │ orchestration, │ ▲ ▲ │ guardrails, LLM │ │ └─ local Chrome (browser_* tools) └──────────────────┘ └───── local shell sandbox (execute_command / execute_code)

The remote agent decides; your machine executes its shell & browser tools and streams results back over the pulse WebSocket. Auth is MasterKey (Authorization: token …, WS ?api_key=); workspaces, threads, history, files, findings, approvals and credits come over a MasterKey-authed REST surface.
